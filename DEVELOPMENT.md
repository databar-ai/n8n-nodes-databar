# Development Guide

This document contains technical details, architectural decisions, and implementation notes for the Databar.ai n8n node.

## Key Technical Decisions

### 1. Absolute vs Relative URLs

**Decision:** Use absolute URLs (`https://api.databar.ai/v1/...`) instead of relative paths.

**Reason:** n8n's `httpRequestWithAuthentication` doesn't automatically apply the `requestDefaults.baseURL` property. Using absolute URLs ensures requests are routed correctly.

**Implementation:**
```typescript
// ❌ Doesn't work
url: '/v1/enrichments/'

// ✅ Works correctly
url: 'https://api.databar.ai/v1/enrichments/'
```

### 2. Async Task Polling

**Decision:** Implement `pollTaskStatus()` as a standalone helper function outside the Databar class.

**Reason:** The function needs to be called from within the `execute` method where `this` refers to `IExecuteFunctions`, not the Databar class instance. Making it a standalone function allows proper access to the execution context.

**Implementation:**
```typescript
// Standalone function before class definition
async function pollTaskStatus(
  context: IExecuteFunctions,
  taskId: string,
  pollInterval: number,
  timeout: number,
): Promise<IDataObject> {
  // ... polling logic
}

// Called from execute method
const completedTask = await pollTaskStatus(this, taskId, pollInterval, timeout);
```

**Configuration:**
- Default poll interval: 5 seconds
- Default timeout: 300 seconds (5 minutes)
- User-configurable via "Additional Options"

### 3. Dynamic Dropdown Loading

**Decision:** Use `loadOptionsMethod` with `searchable: true` for enrichments, waterfalls, and tables.

**Reason:** Provides better UX by allowing users to search and select from available options rather than memorizing IDs.

**Implementation:**
```typescript
// In property definition
{
  displayName: 'Enrichment',
  name: 'enrichmentId',
  type: 'options',
  typeOptions: {
    loadOptionsMethod: 'getEnrichments',
    searchable: true,  // Enables client-side search
  },
  // ...
}

// In methods.loadOptions
async getEnrichments(this: ILoadOptionsFunctions): Promise<INodePropertyOptions[]> {
  const enrichments = await this.helpers.httpRequestWithAuthentication.call(/*...*/);
  return enrichments.map(e => ({
    name: e.name,
    value: e.id,
    description: `${e.description} | ${e.data_source} · ${e.price} Credits`,
  }));
}
```

**Fallback:** "Enrichment Selection" mode allows switching between "From List" and "By ID" if API loading fails.

### 4. Type Safety and Validation

**Decision:** Explicitly validate and convert enrichment IDs from string to number.

**Reason:** The dropdown returns numbers, but they may be received as strings in some contexts. Explicit conversion prevents "Invalid URL" errors.

**Implementation:**
```typescript
const enrichmentIdRaw = this.getNodeParameter('enrichmentId', i);
const enrichmentId = typeof enrichmentIdRaw === 'string' 
  ? parseInt(enrichmentIdRaw, 10) 
  : enrichmentIdRaw;

if (!enrichmentId || isNaN(enrichmentId as number)) {
  throw new NodeOperationError(
    this.getNode(),
    'Please provide a valid enrichment ID',
    { itemIndex: i },
  );
}
```

### 5. Parameter Handling

**Decision:** Use JSON input for enrichment/waterfall parameters instead of fixed collections.

**Reason:** Databar enrichments have dynamic parameters that vary by enrichment. JSON provides flexibility without requiring enrichment-specific UI configuration.

**User Experience:**
```json
// Single enrichment
{"email": "test@example.com", "company": "example.com"}

// Bulk enrichment
[
  {"email": "john@example.com"},
  {"email": "jane@example.com"}
]
```

## Common Issues and Solutions

### Issue 1: "Invalid URL" Error

**Cause:** Using relative URLs like `/v1/enrichments/`

**Solution:** Changed all URLs to absolute: `https://api.databar.ai/v1/enrichments/`

**Fixed in:** Multiple commits, final fix includes template literals like `` `/v1/enrichments/${id}` ``

### Issue 2: "pollTaskStatus is not a function"

**Cause:** `pollTaskStatus` was a class method but couldn't be accessed from `execute` method where `this` is `IExecuteFunctions`.

**Solution:** Moved `pollTaskStatus` outside the class as a standalone helper function.

**Fixed in:** Commit 9d9f92a

### Issue 3: Node Not Appearing in n8n

**Cause:** When installing via `npm install -g`, the `dist` folder wasn't present (GitHub doesn't include build artifacts).

**Solution:** 
1. Build locally: `npm run build`
2. Copy to `~/.n8n/custom/`: `cp -r dist/* ~/.n8n/custom/`
3. Manually add to database if needed:
   ```sql
   INSERT INTO installed_packages VALUES ('n8n-nodes-databar','0.1.0',NOW(),NOW());
   INSERT INTO installed_nodes VALUES ('n8n.nodes.Databar','databar','0.1.0','n8n-nodes-databar',NOW(),NOW());
   INSERT INTO installed_nodes VALUES ('credentials.DatabarApi','databarApi','0.1.0','n8n-nodes-databar',NOW(),NOW());
   ```
4. Restart n8n

### Issue 4: Enrichment Dropdown Not Loading

**Cause:** API request failing or credentials not configured.

**Solution:** 
- Implemented "Enrichment Selection" mode with fallback to "By ID"
- Added try-catch with silent failure for better UX
- Users can manually enter ID if dropdown fails

## API Integration Details

### Authentication
- **Method:** API Key via `x-apikey` header
- **Credential Type:** `databarApi` (defined in DatabarApi.credentials.ts)
- **Test Endpoint:** `GET /v1/user/me`

### Base URL
- **Production:** `https://api.databar.ai`
- **All endpoints:** Prefixed with `/v1/`

### Async Operations

Enrichment and Waterfall `run` and `bulkRun` operations follow this pattern:

1. **Initial Request:**
   - Returns: `{ task_id: "abc123", status: "processing" }`

2. **Polling (if enabled):**
   - Check: `GET /v1/tasks/{task_id}`
   - Every 5 seconds (configurable)
   - Until: `status === 'completed'` or `status === 'failed'`
   - Max wait: 300 seconds (configurable)

3. **Response:**
   - Completed: Returns full task data with results
   - Failed: Throws `NodeOperationError` with error message
   - Timeout: Throws `NodeOperationError` with timeout message

### Rate Limiting
- Not currently implemented in the node
- Databar API may have rate limits - consult Databar documentation

## Future Enhancements

### Potential Improvements

1. **Webhook Support:**
   - Instead of polling, allow Databar to call back when task completes
   - Would reduce API calls and improve efficiency

2. **Batch Operations:**
   - Process multiple enrichments in a single node execution
   - Aggregate results intelligently

3. **Caching:**
   - Cache enrichment list to reduce API calls
   - Invalidate on schedule or manual trigger

4. **Parameter Validation:**
   - Fetch enrichment schema and validate parameters before submission
   - Provide better error messages for invalid inputs

5. **Progress Indication:**
   - Show polling progress in n8n UI
   - Display estimated completion time

6. **Auto-retry:**
   - Implement exponential backoff for failed requests
   - Configurable retry attempts

7. **Streaming:**
   - For bulk operations, stream results as they complete
   - Don't wait for entire batch to finish
